# Note Thumbnail Maker - 要件定義書

## 1. 概要

### 1.1 プロジェクト名
Note Thumbnail Maker

### 1.2 目的
note の記事編集画面で、サムネイル挿入操作を補助する Chrome 拡張を作る。

### 1.3 MVP のゴール
note の記事編集画面上で既存の `画像を追加` ボタンの右側に `タイトルからサムネイルを生成` ボタンを追加し、そのボタンを押したら固定画像をサムネイルとして挿入できること。

## 2. 前提

### 2.1 対象ページ
- 対象 URL は `https://editor.note.com/notes/*/edit/` 形式のみ
- 例: `https://editor.note.com/notes/n22322f42b660/edit/`
- 上記以外のページでは動作しない

### 2.2 UI 方針
- ポップアップは作らない
- 拡張アイコンのクリックは操作起点にしない
- note の記事編集画面内にボタンを差し込んで使う

### 2.3 MVP の割り切り
- ボタン文言は `タイトルからサムネイルを生成` だが、MVP ではタイトル文字列から画像生成は行わない
- MVP では、ボタン押下時に事前に用意した固定画像をサムネイルとして挿入する
- 動作確認を優先し、まずは note エディタ上で実際に挿入完了まで到達することを目標とする

## 3. 機能要件

### 3.1 ボタン表示
- note 記事編集画面でのみ拡張機能を動作させる
- 既存の `画像を追加` ボタンを検出する
- その右側に新しいボタンを 1 つ配置する
- 新しいボタンの表示文言は `タイトルからサムネイルを生成` とする
- 同じボタンが重複表示されないようにする
- ボタンは既存ボタンの右に横並びで配置する
- 見た目は既存 UI から大きく逸脱しないようにする

### 3.2 起動方法
- ユーザーが追加された `タイトルからサムネイルを生成` ボタンをクリックしたときに処理を開始する
- 自動実行はしない

### 3.3 サムネイル挿入
- ボタン押下時、固定のテンプレート画像を用意する
- その画像を note エディタのサムネイルとして挿入する
- 挿入先は note の既存画像追加 UI を利用する
- MVP では、`画像を追加` を押したあとに表示される `画像をアップロード` 導線を利用して画像を渡す
- ユーザーが最終的に記事編集画面上でサムネイルが設定されたことを確認できる状態にする

### 3.4 固定画像仕様
- 画像サイズは `1920 × 1006` ピクセルとする
- MVP では、ユーザーが提示したレイアウト画像を固定画像として扱う
- 画像内容の動的変更は行わない
- タイトル文字列の描画や差し替えは行わない

### 3.5 動作フロー
1. ユーザーが `https://editor.note.com/notes/*/edit/` の記事編集画面を開く
2. 拡張機能が `画像を追加` ボタンの右側に `タイトルからサムネイルを生成` ボタンを表示する
3. ユーザーが追加ボタンを押す
4. 拡張機能が note の既存アップロード導線に固定画像を渡す
5. 記事編集画面上でサムネイル設定済みの状態になる

## 4. 非機能要件

### 4.1 対応ブラウザ
- Google Chrome
- Chromium 系ブラウザ

### 4.2 パフォーマンス
- 対象ページ表示後、ユーザーが待たされない範囲でボタンが表示されること
- ボタン押下後、過度な待ち時間なくサムネイル挿入処理が進むこと

### 4.3 信頼性
- 対象外ページでは何もしない
- `画像を追加` ボタンを検出できない場合は無理に処理しない
- note 側の DOM 構造変更で対象要素が見つからない場合でも、ページ全体を壊さない

### 4.4 セキュリティ
- MVP では外部サーバーへの画像生成リクエストを行わない
- 画像は拡張機能内の固定アセットとして扱う、またはローカル生成データとして扱う
- note へのサムネイル挿入に必要な範囲を超えるデータ取得は行わない

## 5. 技術要件

### 5.1 Manifest
- Chrome Extension Manifest V3 を採用する

### 5.2 想定構成
- `manifest.json`
- `content.js`
  - 対象 URL 判定
  - `画像を追加` ボタンの検出
  - 追加ボタンの差し込み
  - クリック時のサムネイル挿入処理
- `README.md`
  - インストール方法
  - 使い方
- 固定画像アセット
  - 1920 x 1006 のテンプレート画像

### 5.3 想定実装方針
- note 編集画面上で動く Content Script 方式を採用する
- DOM 監視が必要な場合は `MutationObserver` を利用する
- サムネイル設定 UI に対して、既存フローに沿って画像を渡す
- 実装方式は以下のいずれかを許容する
  1. 固定画像ファイルを `File` オブジェクトとしてアップロード入力に渡す
  2. 固定画像データを `Blob` 化してアップロード入力に渡す

### 5.4 必要権限
- note 編集画面で Content Script を実行できること
- `https://editor.note.com/*` に対する対象ページアクセス権限を持つこと

## 6. UX 仕様

### 6.1 ボタン配置
- 既存 UI を壊さない位置に配置する
- 追加ボタンは既存の `画像を追加` ボタンの右側に横並びで配置する
- 既存 UI に寄せた見た目にする
- 極端に目立つ装飾や独自レイアウトは避ける

### 6.2 ユーザー体験
- ユーザーは記事編集画面から離れずに操作できる
- 拡張機能の存在を意識するのは追加ボタンを押すときだけにする
- MVP 時点では細かな設定画面は持たない

## 7. 対象外

以下は MVP では対応しない。

- 記事タイトルを読み取って画像を自動生成する機能
- 生成画像上に動的に文字を描画する機能
- 複数テンプレートの切り替え
- ポップアップ UI
- オプション画面
- 色やレイアウトのカスタマイズ
- 画像保存機能
- タイトル以外の本文内容を使った画像生成
- 外部 AI サービスや画像生成 API との連携

## 8. 未確定事項

- 固定画像を拡張機能内のファイルとして持つか、スクリプト内で生成するか
- ボタン押下中および完了時のフィードバック表示方法
